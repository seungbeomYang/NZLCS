05New Blog section New

A full blog has been added at /blog. It's designed so the team can publish articles (project write-ups, advice posts, news) directly from Sanity, without touching code.

Blog list page /blog

  • 3-column responsive grid of blog cards.
  • Filter chips at the top to filter by category.
  • Each card shows the cover image, category, read time, title and excerpt.
  • Re-fetches every 60 seconds (ISR), so new posts appear quickly.

Blog detail page /blog/[slug]

  • Cover image, title, tags (category + extras), publish date, author, read time.
  • Rich-text body — headings, blockquotes, links, inline images.
  • "Related articles" strip showing 3 other posts.
  • Statically generated for every published post for fast loads.

07Sanity CMS — schemas & setup New

Sanity is the "back office" of the site — a hosted content database with an in-browser editor we host alongside the site itself.

Where to find it

  • Studio URL: https://<your-domain>/studio
  • Local dev: http://localhost:3000/studio
  • Log in with the email/Google account that's been invited to the Sanity project.

The three schemas

Schema

post

One entry per blog article. Drives /blog.

Schema

project

One entry per gallery project. Drives /gallery.

Schema

category

Shared taxonomy with an appliesTo field — the same category can be available to posts, projects, or both.

Dev

Live previews + ISR

Every page is fetched server-side with a 60-second revalidate window, plus tagged caching, so updates appear quickly without redeploying.

08How to add a Blog Post

For non-technical team members. You only need a browser.

  1. Open the Studio. Go to /studio on the live site (or localhost:3000/studio in development) and log in.
  2. Click "Post" in the left sidebar and then the green "+" button to create a new post.
  3. Fill in the title. The slug (URL) is generated automatically from the title — you can edit it if needed.
  4. Write a short excerpt. 1–2 sentences. This is what shows on the blog grid card and in search engine results.
  5. Pick a category. Choose an existing one from the dropdown, or create a new one (see Section 10).
  6. Add tags if you want extra labels on the article (optional).
  7. Set the publish date and confirm the author (defaults to "NZLCS Team").
  8. Set the read time in minutes — used as a hint to readers.
  9. Upload a cover image. Drag-drop in the cover image field. Use the hotspot tool to choose what stays visible when the image is cropped.
  10. Write the body. Use the Portable Text editor — you can add headings, bold/italic, blockquotes, links and inline images.
  11. Click Publish in the bottom-right. The article will appear on the live /blog page within roughly a minute.
Tip: use "Save as draft" while you're still working. Drafts don't appear on the public site.

09How to add a Gallery Project

  1. Open the Studio at /studio and click "Project" in the left sidebar.
  2. Click "+" to create a new project entry.
  3. Title. e.g. "Glenfield Warehouse Re-Coat". The slug is auto-generated.
  4. Pick a category, year, and location (e.g. category: Coatings, year: 2026, location: Auckland).
  5. Write a one-line summary for the card view.
  6. Upload the Before image — the original "before" photo. This will be the left side of the slider.
  7. Upload the After image — the finished "after" photo. This will be the right side. Tip: the two photos should be taken from a similar angle for the slider effect to work well.
  8. Add specs — click "Add item" and enter label/value pairs (e.g. "Surface area" → "1,200 m²", "Duration" → "3 weeks"). Add as many as you want.
  9. Add the project write-up in the body field — describe the brief, the work done, the result.
  10. Add more photos (optional). Each item is an image plus a short label/caption (e.g. "Detail of the parapet flashing").
  11. Publish. The project appears on /gallery within ~60 seconds.
Image guidance: JPG or PNG, ideally 2000px wide or larger. Sanity will automatically resize and serve smaller versions to visitors via its CDN.

10Categories & filtering

Categories are shared between blog posts and gallery projects. A single category document has an appliesTo field — tick "post", "project", or both — which controls where it shows up.

Adding a category

  1. Studio → Category → + to create a new category.
  2. Title (e.g. "Roof Coatings"). The slug is auto-generated.
  3. Set appliesTo — tick "post" if it should appear as a blog filter, "project" for a gallery filter, or both.
  4. (Optional) sort order — a number that controls the order categories appear in the filter bar.
  5. Publish. The new category instantly becomes selectable inside Post and Project documents.
Why this matters: the same "Coatings" or "Concrete" label can be used on a blog article and on a gallery project — visitors see consistent vocabulary across the site, and you only have to maintain one list.

Caching strategy

Both list and detail pages are server-rendered with revalidate: 60. Each fetch is also tagged (e.g. posts, projects, post:<slug>) via sanity/lib/fetch.ts so we can extend to webhook-based instant revalidation later without changing call sites.

12Running locally & environment

Required environment variables

NEXT_PUBLIC_SANITY_PROJECT_ID=xxxxxxxx
NEXT_PUBLIC_SANITY_DATASET=production

Put these in .env.local at the repo root. Both are read by sanity/env.ts.

Common commands

CommandWhat it does
npm installInstall dependencies
npm run devStart the dev server at localhost:3000 (Studio at /studio)
npm run buildProduction build — compiles all blog & gallery pages
npm run startRun the production build locally
npm run lintLint the codebase
Heads-up: on first install, the Sanity Studio dependency tree is large — give the install a couple of minutes the first time. Subsequent runs are fast.
